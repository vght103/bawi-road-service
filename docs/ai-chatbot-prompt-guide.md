## 1. 시스템 프롬프트 구조

### 위치

`supabase/functions/ai-chat/index.ts` 파일의 `buildSystemPrompt()` 함수 내부에서 정의됨.

```typescript
function buildSystemPrompt(academySummary: string): string {
  return `당신은 "바위로드 AI 상담 어시스턴트"입니다. 필리핀 어학연수 전문 상담사 역할을 합니다.

  ## 규칙
  [여기에 규칙이 들어감]

  ## 사용 가능한 어학원 목록
  [DB에서 자동 주입되는 어학원 요약]

  ## 응답 형식
  [JSON 형식 제약]
  `;
}
```

### 구조 세부사항

#### 1. 규칙 섹션

AI의 행동을 지정하는 번호 매겨진 규칙 리스트. 각 규칙은 조건 → 행동 → 예시 패턴을 따름.

**현재 규칙 (v2):**

1. **한국어 답변**: 모든 응답을 한국어로 제공
2. **서비스 정의**: 바위로드는 필리핀 어학연수 중개 서비스
3. **영역 제한**: 어학원/어학연수 관련 질문만 답변, DB 정보 기반
4. **비용 질문 처리**: 구체적 가격 계산 금지. 서버가 가격 키워드 감지 시 자동으로 CTA 전송 (AI는 말만)
5. **모호한 질문**: 먼저 선호사항을 물어봐야 함 (되물어보기)
6. **어학원 검색**: 사용자 메시지에서 키워드를 감지하면, 바로 검색 조건을 파악하세요. 서버가 `extractSearchParams()` 호출해서 검색
7. **결과 없음**: 안내 메시지만 제공 (하단 고정 CTA가 자동으로 표시됨)
8. **톤**: 친근하고 전문적, 간결하게 (200자 이내 권장)

#### 2. 사용 가능한 어학원 목록 섹션

DB의 `academies` 테이블에서 자동으로 조회되는 어학원 요약:

```
- SMEAG Capital (ID: 1) | 세부 | 스파르타 | Tags: ESL, IELTS
- EV Academy (ID: 2) | 마닐라 | 세미스파르타 | Tags: ESL, 비즈니스
- Bright Academy (ID: 3) | 바기오 | 자율형 | Tags: 스피킹
```

**자동 주입 방식**:

- `getAcademySummary()` 함수가 5분 캐시로 조회
- 매 API 요청 시 최신 데이터 기반 프롬프트 생성
- 새 어학원 추가/수정 시 캐시 만료(5분) 후 자동 반영

#### 3. 출력 형식 섹션

**v2 변경:** AI는 일반 텍스트로 답변만 출력합니다. JSON/마크다운 형식화 없이.

예:

```
세부 지역의 스파르타 어학원을 추천드릴게요!
강도 높은 학습 환경과 1:1 수업이 특징입니다.
```

**컴포넌트는 서버가 결정:**

- `academy_cards`: 사용자 메시지의 검색 키워드 매칭 시 서버가 `extractSearchParams()` 호출 → 검색 결과 → SSE `components` 이벤트로 전송
- `cta_button`: 가격 관련 키워드 감지 시 서버가 자동 전송 (AI가 생성하지 않음)
- 하단 고정 CTA: 프론트엔드 `STATIC_CTA` (모든 채팅 화면에서 항상 표시)

### Step 4: 변경 내용 테스트

## 3. 효과적인 규칙 작성 패턴

### 패턴 1: 조건 → 행동 → 예시

**구조:**

```
**[상황]**: [정확한 조건]. [해야 할 행동].
- 예시: "[사용자 입력]" → "[AI 응답]"
```

**좋은 예:**

```
**모호한 질문 → 되물어보기**: 사용자가 구체적인 조건 없이 "어학원 추천해줘" 또는
"필리핀 어디 좋아?"라고 물어보면, 선호사항을 먼저 파악하세요.
- 예시: "필리핀 어학원 추천해줘" → "선호하시는 학습 스타일이 있나요?
  스파르타(집중), 세미스파르타(균형), 자율형 중 어떤 스타일을 원하세요?"
```

**나쁜 예:**

```
**어학원 추천**: 적절하게 추천하세요. (너무 모호함)
```

### 패턴 2: 금지/제한 규칙

금지할 행동을 명확히 하고, **대체 행동을 반드시 명시**:

**좋은 예:**

```
**절대 구체적인 가격을 계산하지 마세요**: 사용자가 "4주 비용 얼마?", "비용 견적" 같은
가격 질문을 하면, 정확한 금액을 계산하거나 안내하지 마세요. 대신,
"정확한 비용은 기간, 코스, 기숙사 옵션에 따라 달라집니다. 무료 견적을 받아보세요!"라고
안내하고 `/quote` 페이지 CTA 버튼을 포함하세요.
- 예시: "4주 숙박 패키지 가격이 얼마지?" → "정확한 비용은 기간, 코스, 기숙사 옵션에 따라
  달라집니다. 무료 견적을 받아보세요!" + 무료 견적 버튼
```

**나쁜 예:**

```
**가격 정보**: 가격을 말하지 마세요. (대체 행동 없음)
```

### 패턴 3: 구체적인 워딩

AI가 명확하게 이해할 수 있도록 **실제 응답 문구를 인용**:

**좋은 예:**

```
"정확한 비용은 기간, 코스, 기숙사 옵션에 따라 달라집니다. 무료 견적을 받아보세요!"
```

**나쁜 예:**

```
비용 정보를 적절하게 처리하세요. (AI가 해석이 애매함)
```

### 패턴 4: 컴포넌트 포함 지시

JSON components가 필요한 경우 명시적으로 작성:

**좋은 예:**

```
**어학원 추천 요청**: 반드시 search_academies 함수를 호출하여 최신 DB 데이터를 기반으로
추천하세요. 결과가 있으면 academy_cards 컴포넌트를 포함하세요.
```

**나쁜 예:**

```
어학원 추천 시 카드를 보여주세요. (구체적인 컴포넌트 이름이 없음)
```

## 4. 테스트 시나리오

프롬프트 수정 후 다음 시나리오로 테스트해야 함.

### 시나리오 1: 모호한 질문 (되물어보기)

**사용자 입력:**

```
어학원 추천해줘
```

**기대 동작:**

- AI는 선호사항 확인 질문만 제공
- 예: "선호하시는 학습 스타일이 있나요? 스파르타(집중), 세미스파르타(균형), 자율형 중..."
- 서버는 검색 키워드 없으므로 academy_cards 미전송
- 하단 고정 CTA는 자동 표시

**검증:**

```bash
# 로컬 함수 요청
curl -X POST http://localhost:54321/functions/v1/ai-chat \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "test-session",
    "message": "어학원 추천해줘"
  }'

# 응답 확인: SSE 스트림에서 message 이벤트만 있고, components 이벤트 없음
```

### 시나리오 2: 구체적 질문 (검색 키워드 감지)

**사용자 입력:**

```
세부 스파르타 어학원 추천
```

**기대 동작:**

- AI는 어학원 소개 텍스트만 출력 (평문)
- 서버가 사용자 메시지에서 "세부", "스파르타" 키워드 감지
- 서버가 `extractSearchParams()` 호출: `{ region: "세부", academy_system: "스파르타" }`
- 검색 결과가 있으면 SSE `components` 이벤트로 `academy_cards` 전송

**검증:**

```bash
curl -X POST http://localhost:54321/functions/v1/ai-chat \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "test-session",
    "message": "세부 스파르타 어학원 추천"
  }'

# 응답 확인:
# 1. SSE 스트림에서 message 이벤트 (AI 답변)
# 2. SSE components 이벤트 (academy_cards) - 서버가 자동 전송
```

### 시나리오 3: 비용 질문 (가격 CTA 자동 전송)

**사용자 입력:**

```
4주 비용 얼마야?
```

**기대 동작:**

- AI는 답변만 제공: "정확한 비용은 기간, 코스, 기숙사 옵션에 따라 달라집니다."
- 서버가 "비용", "얼마", "가격" 키워드 감지 (priceKeywords 배열)
- 서버가 자동으로 SSE `components` 이벤트에 `cta` 타입 전송
- 프론트엔드가 CTA 버튼 렌더링 (무료 견적 받기 → /quote)

**검증:**

```bash
curl -X POST http://localhost:54321/functions/v1/ai-chat \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "test-session",
    "message": "4주 비용 얼마야?"
  }'

# 응답 확인:
# 1. SSE message 이벤트에 "정확한 비용은..." 텍스트
# 2. SSE components 이벤트에 type: "cta" 전송됨 (서버 자동)
```

### 시나리오 4: 무관한 질문 (거절 + 고정 CTA)

**사용자 입력:**

```
오늘 날씨 어때?
```

**기대 동작:**

- AI는 정중한 거절 메시지만 제공: "저는 필리핀 어학연수 전문 상담 AI입니다. 어학연수 관련 질문을 해주세요!"
- 서버는 검색 키워드 없으므로 academy_cards 미전송
- 가격 키워드도 없으므로 cta 미전송
- 하단 고정 CTA는 자동 표시

**검증:**

```bash
curl -X POST http://localhost:54321/functions/v1/ai-chat \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "test-session",
    "message": "오늘 날씨 어때?"
  }'

# 응답 확인:
# 1. SSE message 이벤트만 있음 (거절 메시지)
# 2. components 이벤트 없음 (프론트엔드 STATIC_CTA만 표시)
```

### 시나리오 5: 검색 결과 없음

**사용자 입력:**

```
마닐라 스파르타 어학원 추천
```

(마닐라에 스파르타 시스템이 없는 경우)

**기대 동작:**

- AI는 답변만 제공: "조건에 맞는 어학원을 찾지 못했습니다. 다른 지역이나 학습 방식을 고려해보세요!"
- 서버가 "마닐라", "스파르타" 키워드 감지
- 서버가 `extractSearchParams()` 호출하지만 검색 결과 `[]` (빈 배열)
- SSE components 이벤트 미전송 (academy_cards 없음)
- 하단 고정 CTA 자동 표시

**검증:**

```bash
# 먼저 academies 테이블 확인: 마닐라 스파르타 없음
select * from academies where region = '마닐라' and academy_system = '스파르타';

# 그 후 테스트
curl -X POST http://localhost:54321/functions/v1/ai-chat \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "test-session",
    "message": "마닐라 스파르타 어학원 추천"
  }'

# 응답 확인:
# 1. SSE message 이벤트 (안내 메시지)
# 2. academy_cards 컴포넌트 미전송
```

## 5. 키워드 기반 검색 (extractSearchParams)

### 개요

**v2 변경:** AI가 함수를 호출하는 대신, 서버가 사용자 메시지에서 키워드를 감지하여 `extractSearchParams()` 함수를 자동으로 호출합니다. AI는 답변만 제공합니다.

### 서버 검색 메커니즘

**1. 서버가 키워드 매칭:**

- 사용자 메시지에서 지역, 학습 시스템, 코스 태그 키워드 추출
- 매칭되는 조건이 있으면 검색 수행

**2. extractSearchParams() 함수 호출:**

```typescript
extractSearchParams(userMessage: string) {
  // 지역, 시스템, 태그 키워드 매칭
  // 반환: { region?, academy_system?, tags? }
}
```

**3. SSE 컴포넌트로 결과 전송:**

- 검색 결과 있음 → `academy_cards` SSE 이벤트
- 검색 결과 없음 → 컴포넌트 미전송
- 가격 키워드 감지 → `cta` SSE 이벤트 (별도)

### 검색 키워드 매칭

#### 지역 (regions)

```javascript
(세부, 세부시, cebu | 바기오, bagui̳o | 클락, clark | 마닐라, manila);
```

#### 학습 시스템 (academy_systems)

```javascript
스파르타 | 세미스파르타 | 자율형;
```

#### 코스 태그 (tags)

```javascript
(ESL, 이에스엘 | IELTS, 아이엘츠 | TOEIC, 토익 | 스피킹, 회화 | 비즈니스);
```

### 검색 호출 조건

**서버가 검색을 수행하는 경우:**

- 사용자 메시지에 위의 키워드 중 **1개 이상** 포함
- 예: "세부" (지역만), "스파르타" (시스템만), "IELTS" (태그만), "세부 스파르타 IELTS" (모두)

**서버가 검색하지 않는 경우:**

- 키워드가 없음
- 예: "어학원 추천해줘", "비용 얼마야?"

### AI의 역할

**v2에서 AI는:**

1. 사용자 질문에 답변만 제공 (평문)
2. 검색 수행 여부는 서버가 결정
3. 컴포넌트 생성하지 않음 (서버가 SSE로 전송)

**규칙 5 (수정된 버전):**

```
5. **어학원 검색**: 사용자가 지역, 학습 시스템, 코스 태그 같은 구체적 조건을 언급하면,
   서버가 자동으로 검색을 수행합니다. 당신은 검색 조건을 파악할 수 있도록 명확한 답변만
   제공하세요. 컴포넌트는 서버가 결정합니다.
```

### 가격 키워드 자동 감지 (priceKeywords)

서버는 별도의 `priceKeywords` 배열로 가격 관련 키워드를 감지:

```javascript
priceKeywords: ["비용", "가격", "얼마", "금액", "수업료", "학비", "패키지", "비용은", "요금", "견적", "가격이"];
```

**가격 키워드 감지 시:**

- AI가 가격 답변을 제공하면
- 서버가 자동으로 `cta` SSE 이벤트 전송
- 프론트엔드가 무료 견적 받기 버튼 렌더링

**AI의 가격 답변 규칙:**

```
4. **비용 질문**: 구체적인 가격 계산이나 금액 제시는 하지 마세요.
   "정확한 비용은 기간, 코스, 기숙사 옵션에 따라 달라집니다."라고만 안내하세요.
   CTA 버튼은 서버가 자동으로 전송합니다.
```

```bash
supabase functions serve ai-chat
```

```bash
supabase functions deploy ai-chat
```

```typescript
const regionKeywords = {
  세부: ["세부", "세부시", "cebu"],
  바기오: ["바기오", "bagui̳o"],
  // ... "회화" 같은 동의어 추가
};

const tagKeywords = {
  스피킹: ["스피킹", "회화", "speaking", "conversation"],
  // ... 누락된 동의어 추가
};
```

### 문제 4: 어학원 이미지가 표시 안 됨

```sql
select id, name, images from academies where images is not null;
```

### 문제 5: SSE 스트림 연결 끊김

```bash
supabase functions serve ai-chat
```

```typescript
const eventSource = new EventSource("/ai-chat?session_id=xxx");
eventSource.addEventListener("message", (e) => {
  console.log("Received:", e.data);
});
eventSource.onerror = (e) => {
  console.error("SSE Error:", e);
};
```

4. **규칙은 AI 응답에만 집중**: 컴포넌트 생성 규칙 제거 ✓ → 검색/CTA 규칙은 서버 레벨에서 관리

5. **테스트 시나리오 5가지 검증**: 프롬프트 변경 후 항상 실행
