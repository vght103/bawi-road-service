# AI 상담 챗봇 컨셉 문서

> **작성일**: 2026-02-27 (v2 수정)
> **상태**: v2 구현 완료

---

## 개요

홈 히어로 섹션의 정적 챗봇 목업을 **실제 AI 챗봇**으로 전환한다.
홈에서는 진입점 역할만 하고, 본격적인 대화는 **별도 채팅 페이지(`/chat`)**에서 진행한다.

---

## 사용자 플로우

```
홈 히어로 챗봇 카드 (input 활성화)
  ↓ 메시지 입력 + 전송 클릭
/chat?q={입력내용} 페이지로 이동
  ↓ 첫 메시지 자동 전송
AI 응답 (텍스트 / 어학원 카드 / CTA 버튼)
```

### 홈 히어로 변경사항

- [x] 플로팅 뱃지 2개 제거 ("1:1 수업 하루 4시간", "세부 직항 4시간")
- [x] input `disabled` 해제 → 활성화
- [x] 전송 버튼 클릭 시 `/chat?q={입력내용}` 으로 라우팅
- [x] 기존 하드코딩된 대화 내용은 유지 (미리보기 역할)

| 사용자 질문 유형 | AI 응답 형태 |

| 일반 상담 질문 | 텍스트 응답 |

## AI 동작 방식

### 시스템 프롬프트 (개요)

- 바위로드 AI 상담 어시스턴트 역할
- 필리핀 어학연수 전문 상담
- 어학원 DB 정보 기반 응답
- **비용 계산은 하지 않음** → 견적 페이지로 유도
- 어학원 추천 요청 시 키워드 기반 사전 검색 (`extractSearchParams`) 수행
- **일반 텍스트만 생성**, components는 서버에서 결정

### 어학원 카드 검색 (개선)

- Function calling 제거 → 키워드 기반 사전 검색으로 변경
- AI는 텍스트만 생성 (예: "세부 스파르타 어학원을 추천드릴게요!")
- 백엔드가 `extractSearchParams`로 조건(지역, 스타일) 추출
- DB 쿼리 후 어학원 카드 컴포넌트를 자동 생성

### 응답 포맷 (SSE 이벤트 프로토콜)

AI 응답을 **SSE (Server-Sent Events)** 스트리밍으로 전달:

```
data: {"type":"text","content":"세부"}
data: {"type":"text","content":" 지역"}
data: {"type":"text","content":" 스파르타"}
data: {"type":"text","content":" 어학원을"}
data: {"type":"text","content":" 추천드릴게요!"}
data: {"type":"components","data":[{"type":"academy_cards","data":[{"id":"smeag","name":"SMEAG Capital","region":"세부","style":"스파르타"}]}]}
data: {"type":"done","session_id":"uuid-1234"}
```

**이벤트 타입:**

| 타입 | 설명 | 예시 |
|------|------|------|
| `text` | 텍스트 토큰 (반복) | 실시간 스트리밍 |
| `components` | 어학원 카드 또는 CTA | 사용자 질문 분석 후 자동 삽입 |
| `cta` | CTA 버튼 (가격 키워드) | 하단 고정 칩 또는 인라인 |
| `done` | 응답 완료 | session_id 포함 |

### 어학원 카드 UI (메시지 버블 내)

v1의 가로 캐러셀에서 세로 리스트로 변경:

```

│ [이미지]  SMEAG Capital │
│           세부 · 스파르타 │
│           [자세히 보기 →] │
├─────────────────────────┤
│ [이미지]  EV Academy    │
│           세부 · 일반     │
│           [자세히 보기 →] │
└─────────────────────────┘
```

구분선으로 카드 분리.

### CTA 버튼

**기본 위치**: 메시지 하단 고정 칩
- 무료 견적 받기 (`/quote`)
- 1:1 상담 신청 (`/inquiry`)

**인라인 CTA**: 가격 키워드 감지 시 메시지 내 자동 삽입

## 세션 관리

- 매 새 채팅 시 UUID 기반 session_id 생성
- sessionStorage에 session_id 저장
- 페이지 새로고침 시 DB에서 해당 세션의 채팅 이력 복원
- 로그인 시 user_id와 session 연결
