## 런칭 전 필수 (BLOCKER)

사업자 등록 및 실제 정보 기입이 완료되어야 법적으로 서비스 운영이 가능합니다.

### B-1. 사업자 등록 + 통신판매업 신고

수속료(10만원)를 받는 서비스이므로 **통신판매업 신고 필수** (전자상거래법).

**절차:**
1. 세무서에서 **개인사업자 등록** (아직 안 된 경우)
2. **정부24 (gov.kr)** 에서 통신판매업 신고
3. 신고 완료 후 받는 번호를 코드에 기입

### B-2. 사업자 정보 placeholder(-)를 실제 값으로 교체

현재 `-`로 되어 있는 항목들을 실제 값으로 교체해야 합니다.

**교체 필요 위치 (3곳):**

| 파일 | 교체할 항목 |
|------|-------------|
| `src/data/legal/terms-of-service.ts` 제14조 | 대표자, 사업자등록번호, 통신판매업 신고번호, 주소, 전화번호 |
| `src/pages/privacy.astro` 제11조 | 성명, 연락처 |
| `src/components/Footer.astro` | 대표, 사업자등록번호, 통신판매업 신고번호 |

> 전자상거래법 제13조: 상호, 대표자, 주소, 전화번호, 사업자등록번호, 통신판매업 신고번호를 표시해야 함. 미표시 시 시정명령 및 과태료.

### B-3. 수속 신청 동의 체크박스 보완

현재 이용약관 + 환불규정 2개만 있음. **법적으로 3개 더 필요:**

```
[ ] (필수) 개인정보 수집·이용에 동의합니다.
[ ] (필수) 개인정보 제3자 제공(어학원)에 동의합니다.
[ ] (필수) 개인정보 국외이전에 동의합니다.
```

**구현 위치:** `src/react-pages/enrollment/` 의 StepTermsAgreement 컴포넌트

**이유:**
- 개인정보보호법: 개인정보 수집/이용, 제3자 제공, 국외이전은 각각 별도 동의 필요
- 여권번호 등 고유식별정보는 별도 동의 대상 (개인정보보호법 제24조)

### B-4. 서비스 도메인 설정

현재 설정이 개발 도메인(`bawi-road-service.pages.dev`)으로 되어 있음.

**교체 필요 위치:**
| 파일 | 내용 |
|------|------|
| `astro.config.mjs` | `site: "https://bawi-road-service.pages.dev"` → 실서비스 도메인 |
| `public/robots.txt` | Sitemap URL |
| `public/sitemap.xml` | 모든 URL의 도메인 |

## 런칭 전 권장 (HIGH)

법적 필수는 아니지만 서비스 운영에 중요한 항목들입니다.

### H-1. 웹 분석(Analytics) 설치

현재 사용자 행동 추적이 전혀 없습니다. 최소 하나는 설치 권장:

- **Google Analytics 4 (GA4)** — 페이지뷰, 전환 추적
- 또는 **Cloudflare Web Analytics** — Cloudflare Pages 사용 중이라 연동 간편
- 추적할 전환: 견적 요청 완료, 상담 신청 완료, 수속 신청 완료

### H-2. `/search` 중복 페이지 정리

`/search`와 `/academies`가 동일한 `AcademySearchPage` 컴포넌트를 사용함.
SEO에서 중복 콘텐츠로 감지될 수 있으므로:
- `/search` 삭제 또는
- `/search` → `/academies`로 리다이렉트

### H-3. React Error Boundary 추가

현재 React 컴포넌트에서 에러 발생 시 전체 페이지가 빈 화면이 됩니다.
`client:only="react"` 아일랜드마다 Error Boundary를 감싸면 에러 시 fallback UI를 보여줄 수 있음.

### H-4. 수속 신청 시 성인 확인

이용약관 제6조의2(미성년자 이용)가 이미 존재하지만, 실제 UI에서 나이 확인이 없음.
About 페이지에서 "성인 전용"이라고 언급하지만 기술적 차단 없음.

**MVP 최소 구현:**
- 수속 신청 시 `[ ] 만 18세 이상임을 확인합니다` 체크박스 1개 추가
- 미성년자 전용 플로우(법정대리인 동의 등)는 추후 구현

### H-5. 출국 준비 정보 등록 시 여권정보 별도 동의

개인정보처리방침 제2조에 여권정보 수집을 명시했으므로, 여권 정보 입력 화면에서:
```
[ ] (필수) 여권번호 등 고유식별정보 수집·이용에 동의합니다.
```
별도 체크박스 필요 (개인정보보호법 제24조).

**구현 위치:** `src/react-pages/enrollment/` 서류 업로드 단계

## 추후 개선 (MEDIUM)

당장 런칭에는 영향 없지만, 서비스 성장 시 필요한 항목들입니다.

### M-1. 쿠키 동의 배너

개인정보처리방침 제10조에서 쿠키 사용을 명시하면서 동의 수단이 없음.
한국법은 EU GDPR보다 규제가 약하지만, 개인정보처리방침과 일관성을 위해 간단한 배너 권장.

### M-2. 구조화된 데이터 (JSON-LD)

검색 결과 노출 개선을 위해 `LocalBusiness`, `Organization` 등의 schema.org 마크업 추가 가능.

### M-3. 미성년자 법정대리인 동의 자동화

현재 이용약관에 조항은 있으나 UI 미구현:
- 회원가입 시 생년월일 기반 미성년자 판별
- 미성년자면 법정대리인 이메일 입력 → 동의 확인 이메일 발송
- MVP에서는 카카오톡 상담으로 수동 확인 가능 (현실적 방안)

### M-4. 서버사이드 폼 검증

클라이언트 검증은 존재하나, 서버(Edge Function) 레벨에서의 입력값 검증 강화 권장.
특히 수속 신청, 견적 요청, 상담 신청 등 중요 폼.
