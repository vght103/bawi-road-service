# 인증 (로그인/회원가입/계정찾기/비밀번호 재설정)

> 상태: 구현완료
> 최종 수정일: 2026-03-11
> 관련 페이지: src/pages/login.astro, src/pages/signup.astro, src/pages/find-account.astro, src/pages/reset-password.astro, src/react-pages/auth/

## 개요

Supabase Auth 기반 사용자 인증 시스템. 이메일/비밀번호 로그인, 회원가입(이메일 인증), 이메일 찾기(이름+핸드폰), 비밀번호 찾기(재설정 링크 발송), 비밀번호 재설정을 지원한다.

## 문제 정의

수속 신청, 마이페이지 등 사용자 식별이 필요한 기능을 이용하려면 로그인이 필요하다. 이메일 기반 인증과 계정 복구 기능을 제공해야 한다.

## 사용자 스토리

- [x] 사용자는 이메일과 비밀번호로 로그인할 수 있다
- [x] 사용자는 이름/핸드폰/이메일/비밀번호로 회원가입할 수 있다
- [x] 사용자는 회원가입 후 이메일 인증을 완료해야 로그인할 수 있다
- [x] 사용자는 이름+핸드폰 번호로 등록된 이메일을 찾을 수 있다
- [x] 사용자는 이름+이메일로 비밀번호 재설정 링크를 받을 수 있다
- [x] 사용자는 재설정 링크를 통해 새 비밀번호를 설정할 수 있다

- REQ-014: 회원가입 시 이름, 핸드폰 번호, 이메일, 비밀번호를 입력받는다

## 상세 스펙

### UI/UX

#### 공통 레이아웃

모든 인증 페이지는 동일한 2분할 레이아웃:
- 좌측: 폼 영역
- 우측: 마스코트 일러스트 + 슬로건 (데스크탑 전용, lg 이상)

#### 로그인 페이지 (/login)

- 이메일 + 비밀번호(표시/숨김 토글) 입력
- 에러 메시지: 빨간 배경 박스
- 하단: 이메일 찾기 | 비밀번호 찾기 | 회원가입 링크

#### 회원가입 페이지 (/signup)

- react-hook-form + zod 기반 유효성 검사
- 필드: 이름, 핸드폰(숫자/하이픈만), 이메일, 비밀번호, 비밀번호 확인
- 비밀번호 강도 체크: 실시간으로 3개 조건(8자/숫자/특수기호) 표시 (충족: 초록, 미충족: 회색)
- 비밀번호 일치 여부: 실시간 표시 (일치: 초록, 불일치: 빨강)
- 완료 시: SignupSuccess 화면 (인증 메일 발송 안내 + 로그인 이동)

#### 계정 찾기 페이지 (/find-account)

- 2개 탭: 이메일 찾기 / 비밀번호 찾기
- `?tab=find-password` 쿼리 파라미터로 탭 초기 선택 가능
- 이메일 찾기: 이름 + 핸드폰 → 마스킹된 이메일 표시 (초록 박스)
- 비밀번호 찾기: 이름 + 이메일 → 재설정 링크 발송 안내

#### 비밀번호 재설정 페이지 (/reset-password)

- 이메일 링크 클릭 후 접근
- 새 비밀번호 + 확인 입력 (강도 체크 + 일치 체크 동일 적용)
- 완료 시: ResetPasswordSuccess 화면

### 데이터 모델

- Supabase Auth (auth.users)
- user_metadata: name, phone (회원가입 시 저장)

### API

- `useAuth` 훅: signIn, signUp, findEmail, sendPasswordResetEmail, changePassword

### 비즈니스 로직

- 로그인 에러 한국어 변환: "Invalid login credentials" → "이메일 또는 비밀번호가 올바르지 않습니다."
- 로그인 에러: "Email not confirmed" → "이메일 인증이 완료되지 않았습니다."
- 회원가입 에러: "already registered" → 이메일 필드에 "이미 등록된 이메일입니다." 표시
- 핸드폰 번호: 입력 시 숫자/하이픈 외 차단, 저장 시 하이픈 제거
- 비밀번호 규칙: 8자 이상 + 숫자 1개 이상 + 특수기호 1개 이상

### 엣지케이스

- 빈 입력 제출: 각 필드별 에러 메시지 표시 (zod 검증 또는 수동 체크)
- 잘못된 이메일 형식: "올바른 이메일 형식을 입력해주세요."
- 잘못된 핸드폰 형식: "올바른 핸드폰 번호를 입력해주세요."
- 이메일 찾기 실패: error 메시지 표시
- 제출 중: LoadingOverlay + 버튼 비활성화
