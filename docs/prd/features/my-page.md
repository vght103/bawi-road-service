## 개요

로그인 사용자의 개인 정보 확인, 수속 목록 조회, 비밀번호 변경, 회원 탈퇴를 제공하는 마이페이지.

## 문제 정의

사용자가 자신의 수속 현황을 한 곳에서 관리하고, 계정 설정(비밀번호 변경, 탈퇴)을 수행할 수 있는 공간이 필요하다.

## 사용자 스토리

- [x] 사용자는 내 정보(이름, 이메일, 연락처)를 확인할 수 있다
- [x] 사용자는 내 수속 목록을 확인하고 상세 페이지로 이동할 수 있다
- [x] 사용자는 비밀번호를 변경할 수 있다
- [x] 사용자는 회원 탈퇴를 할 수 있다
- [x] 사용자는 로그아웃할 수 있다

## 상세 스펙

### UI/UX

#### 비로그인 상태

- 로그인 유도 메시지 + "로그인하기" 버튼 (`/login?from=/my`)

#### 섹션 1 — 내 정보

- 이름 / 이메일 / 연락처 표시
- members 테이블 미등록 시 이름/연락처에 "-" 표시
- 로그아웃 버튼

#### 섹션 2 — 내 수속

- 수속 목록 카드: 어학원명, 상태 뱃지(색상별), 코스/기숙사/기간, 시작일
- 각 카드 클릭 → `/enrollment/:id`
- 수속 없으면 빈 상태 + "새 수속 신청" CTA → `/enrollment/apply?from=my-page`

#### 섹션 3 — 비밀번호 변경

- 새 비밀번호 + 비밀번호 확인 입력
- 실시간 유효성 체크 (8자 이상, 숫자, 특수기호, 일치)
- 성공/에러 인라인 메시지

#### 섹션 4 — 회원 탈퇴

- 버튼 클릭 → 입력 영역 노출
- "회원탈퇴" 정확히 입력 → 확인 버튼 활성화
- 탈퇴 완료 → 로그아웃 + 홈 이동

### 데이터 모델

- `auth.users` — 이메일, user_id
- `members` 테이블 — name, phone, role
- `enrollments` 테이블 — 수속 목록 (enrollment.md 참조)

### API / 상태 관리

- `useAuth()` 훅 — user, session, loading, signOut, changePassword, deleteAccount
- `useMember()` 훅 — member (name, phone), isLoading
- `fetchMyEnrollments(userId)` — React Query 직접 사용 (커스텀 훅 없음)
- `deleteAccount()` — `supabase.rpc("delete_own_account")` 호출

### 비즈니스 로직

- ADMIN 역할 계정은 signIn 시 즉시 로그아웃 + 에러 표시 (서비스 접근 차단)
- `$loading`이 true인 동안 auth 체크 완료까지 UI 렌더링 대기
- 비밀번호 규칙: 8자 이상 + 숫자 1개 + 특수기호 1개
- 회원 탈퇴: Supabase RPC `delete_own_account` → 서버 측 사용자 데이터 삭제

### 엣지케이스

- auth 로딩 중: 로딩 상태 표시 (빈 화면 방지)
- members 테이블에 행이 없는 경우: 이름/연락처 "-" 표시
- 수속 0건: 빈 상태 UI + CTA
- 비밀번호 변경 중/탈퇴 중: LoadingOverlay 표시
- 탈퇴 확인 텍스트 불일치: 확인 버튼 비활성화
